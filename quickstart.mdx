<Note>

</Note>

## Prerequisites

You'll need:
- Docker installed (or see [Installation](/installation) for other methods)
- A WhatsApp account (phone number)

## Quick Start with Docker

<Steps>
  <Step title="Run the container">
    Start the GOWA API server using Docker:

    <CodeGroup>
    ```bash Docker Hub
    docker run --detach \
      --publish=3000:3000 \
      --name=whatsapp \
      --restart=always \
      --volume=$(docker volume create --name=whatsapp):/app/storages \
      aldinokemal2104/go-whatsapp-web-multidevice rest
    ```

    ```bash GitHub Container Registry
    docker run --detach \
      --publish=3000:3000 \
      --name=whatsapp \
      --restart=always \
      --volume=$(docker volume create --name=whatsapp):/app/storages \
      ghcr.io/aldinokemal/go-whatsapp-web-multidevice rest
    ```
    </CodeGroup>

    The API will be available at `http://localhost:3000`
  </Step>

  <Step title="Open the web interface">
    Navigate to `http://localhost:3000` in your browser. You'll see the GOWA homepage.

    <Frame>
      <img src="https://raw.githubusercontent.com/aldinokemal/go-whatsapp-web-multidevice/main/gallery/homepage.png" alt="GOWA Homepage" />
    </Frame>
  </Step>

  <Step title="Connect your WhatsApp account">
    Click **Login** or navigate to the login page:

    <Tabs>
      <Tab title="QR Code (Recommended)">
        1. Navigate to `http://localhost:3000/app/login`
        2. A QR code will appear on the screen
        3. Open WhatsApp on your phone:
           - **Android**: Tap ⋮ → **Linked devices** → **Link a device**
           - **iOS**: Tap **Settings** → **Linked Devices** → **Link a Device**
        4. Scan the QR code displayed on your screen
        5. Wait for "Connected" status confirmation

        <Frame>
          <img src="https://raw.githubusercontent.com/aldinokemal/go-whatsapp-web-multidevice/main/gallery/login.png" alt="QR Code Login" />
        </Frame>
      </Tab>

      <Tab title="Pairing Code">
        1. Navigate to `http://localhost:3000/app/login-with-code`
        2. Enter your phone number (e.g., `628123456789`)
        3. Click **Get Pairing Code**
        4. A pairing code will be displayed
        5. Open WhatsApp on your phone and go to **Linked Devices**
        6. Tap **Link a device** → **Link with phone number instead**
        7. Enter the pairing code shown on screen
        8. Wait for "Connected" status confirmation

        <Frame>
          <img src="https://raw.githubusercontent.com/aldinokemal/go-whatsapp-web-multidevice/main/gallery/login-with-code.png" alt="Pairing Code Login" />
        </Frame>
      </Tab>
    </Tabs>

    <Note>
    Your WhatsApp session is stored in the Docker volume and persists across restarts.
    </Note>
  </Step>

  <Step title="Send your first message">
    Now that you're connected, send a test message using the API:

    <CodeGroup>
    ```bash cURL
    curl -X POST http://localhost:3000/send/message \
      -H "Content-Type: application/json" \
      -d '{
        "phone": "628123456789",
        "message": "Hello from GOWA! 🚀"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('http://localhost:3000/send/message', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        phone: '628123456789',
        message: 'Hello from GOWA! 🚀'
      })
    });

    const result = await response.json();
    console.log(result);
    ```

    ```python Python
    import requests

    response = requests.post(
        'http://localhost:3000/send/message',
        json={
            'phone': '628123456789',
            'message': 'Hello from GOWA! 🚀'
        }
    )

    print(response.json())
    ```

    ```php PHP
    $response = file_get_contents(
        'http://localhost:3000/send/message',
        false,
        stream_context_create([
            'http' => [
                'method' => 'POST',
                'header' => 'Content-Type: application/json',
                'content' => json_encode([
                    'phone' => '628123456789',
                    'message' => 'Hello from GOWA! 🚀'
                ])
            ]
        ])
    );

    $result = json_decode($response, true);
    print_r($result);
    ```
    </CodeGroup>

    **Expected Response:**
    ```json
    {
      "code": "SUCCESS",
      "message": "Message sent successfully",
      "results": {
        "message_id": "3EB0C127D7BACC83D6A1",
        "status": "success"
      }
    }
    ```

    <Tip>
    Phone numbers should be in international format without `+` or spaces (e.g., `628123456789` for Indonesia, `14155552671` for US).
    </Tip>
  </Step>
</Steps>

## Common Use Cases

<AccordionGroup>
  <Accordion title="Send images with captions">
    ```bash
    curl -X POST http://localhost:3000/send/image \
      -F "phone=628123456789" \
      -F "image=@/path/to/image.jpg" \
      -F "caption=Check this out!"
    ```
  </Accordion>

  <Accordion title="Send to multiple recipients">
    Send the same message to multiple phone numbers:
    ```bash
    for phone in 628123456789 628987654321 621234567890; do
      curl -X POST http://localhost:3000/send/message \
        -H "Content-Type: application/json" \
        -d '{"phone": "'$phone'", "message": "Bulk message"}'
    done
    ```
  </Accordion>

  <Accordion title="Check connection status">
    ```bash
    curl http://localhost:3000/app/status
    ```

    Response:
    ```json
    {
      "code": "SUCCESS",
      "message": "Connection status retrieved",
      "results": {
        "is_connected": true,
        "is_logged_in": true,
        "device_id": "628123456789@s.whatsapp.net"
      }
    }
    ```
  </Accordion>

  <Accordion title="Send messages to groups">
    Groups use the `@g.us` suffix:
    ```bash
    curl -X POST http://localhost:3000/send/message \
      -H "Content-Type: application/json" \
      -d '{
        "phone": "120363402106123456@g.us",
        "message": "Hello group!"
      }'
    ```

    <Tip>
    Get your group JIDs by calling `GET /user/my/groups`
    </Tip>
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="QR code not appearing">
    - Check that Docker container is running: `docker ps`
    - Check logs: `docker logs whatsapp`
    - Ensure port 3000 is not in use by another application
    - Try restarting the container: `docker restart whatsapp`
  </Accordion>

  <Accordion title="Connection lost after scan">
    - Check your internet connection
    - Ensure your phone has a stable internet connection
    - Try reconnecting: `curl http://localhost:3000/app/reconnect`
    - Check device status: `curl http://localhost:3000/app/status`
  </Accordion>

  <Accordion title="Message not sending">
    - Verify the recipient number format (no `+`, spaces, or hyphens)
    - Ensure you're connected: check `http://localhost:3000/app/status`
    - Check if the number is registered on WhatsApp
    - Review logs for errors: `docker logs whatsapp`
  </Accordion>

  <Accordion title="Port 3000 already in use">
    Change the port mapping when starting Docker:
    ```bash
    docker run --detach \
      --publish=8080:3000 \
      --name=whatsapp \
      --restart=always \
      --volume=$(docker volume create --name=whatsapp):/app/storages \
      aldinokemal2104/go-whatsapp-web-multidevice rest
    ```
    Then access the API at `http://localhost:8080`
  </Accordion>
</AccordionGroup>

- Adding [basic authentication](/guides/configuration#basic-authentication)

- Setting up proper logging and monitoring
