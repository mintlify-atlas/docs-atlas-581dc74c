GOWA (Go WhatsApp Web Multi-Device API) can be installed in multiple ways depending on your environment and requirements.

## System Requirements

<CardGroup cols={2}>
  <Card title="Platform Support" icon="laptop">
    - Linux (x86_64, ARM64)
    - macOS (Intel, Apple Silicon)
    - Windows (x86_64 via WSL)
  </Card>

  <Card title="Dependencies" icon="cube">
    - **Docker** (recommended) or
    - **Go 1.24.0+** (for building from source)
    - **FFmpeg** (for media processing)
  </Card>
</CardGroup>

<Note>
Docker is the **recommended** installation method as it includes all dependencies and works consistently across platforms.
</Note>

## Installation Methods

<Tabs>
  <Tab title="Docker (Recommended)">
    ### Docker Installation

    Docker is the simplest and most reliable way to run GOWA. All dependencies (FFmpeg, libwebp) are included.

    <Steps>
      <Step title="Pull the image">
        Choose either Docker Hub or GitHub Container Registry:

        <CodeGroup>
        ```bash Docker Hub
        docker pull aldinokemal2104/go-whatsapp-web-multidevice:latest
        ```

        ```bash GitHub Container Registry
        docker pull ghcr.io/aldinokemal/go-whatsapp-web-multidevice:latest
        ```
        </CodeGroup>
      </Step>

      <Step title="Create a volume for persistent storage">
        Create a Docker volume to store WhatsApp sessions and media:

        ```bash
        docker volume create whatsapp
        ```

        <Tip>
        This volume persists your WhatsApp session across container restarts.
        </Tip>
      </Step>

      <Step title="Run the container">
        <CodeGroup>
        ```bash Basic (Docker Hub)
        docker run --detach \
          --publish=3000:3000 \
          --name=whatsapp \
          --restart=always \
          --volume=whatsapp:/app/storages \
          aldinokemal2104/go-whatsapp-web-multidevice rest
        ```

        ```bash Basic (GitHub Container Registry)
        docker run --detach \
          --publish=3000:3000 \
          --name=whatsapp \
          --restart=always \
          --volume=whatsapp:/app/storages \
          ghcr.io/aldinokemal/go-whatsapp-web-multidevice rest
        ```

        ```bash With Options
        docker run --detach \
          --publish=3000:3000 \
          --name=whatsapp \
          --restart=always \
          --volume=whatsapp:/app/storages \
          aldinokemal2104/go-whatsapp-web-multidevice rest \
          --basic-auth=admin:admin \
          --port=3000 \
          --debug=true \
          --os=Chrome \
          --autoreply="Please wait, I'll respond shortly"
        ```
        </CodeGroup>
      </Step>

      <Step title="Verify it's running">
        Check container status:
        ```bash
        docker ps
        ```

        Test the API:
        ```bash
        curl http://localhost:3000/app/devices
        ```

        View logs:
        ```bash
        docker logs whatsapp
        ```
      </Step>
    </Steps>

    ### Docker Management Commands

    <AccordionGroup>
      <Accordion title="Stop the container">
        ```bash
        docker stop whatsapp
        ```
      </Accordion>

      <Accordion title="Start the container">
        ```bash
        docker start whatsapp
        ```
      </Accordion>

      <Accordion title="Restart the container">
        ```bash
        docker restart whatsapp
        ```
      </Accordion>

      <Accordion title="View logs">
        ```bash
        # Follow logs in real-time
        docker logs -f whatsapp

        # View last 100 lines
        docker logs --tail=100 whatsapp
        ```
      </Accordion>

      <Accordion title="Remove container and start fresh">
        ```bash
        # Stop and remove container
        docker stop whatsapp
        docker rm whatsapp

        # Optionally remove the volume (THIS DELETES YOUR SESSION)
        docker volume rm whatsapp

        # Start a new container
        docker run --detach \
          --publish=3000:3000 \
          --name=whatsapp \
          --restart=always \
          --volume=$(docker volume create --name=whatsapp):/app/storages \
          aldinokemal2104/go-whatsapp-web-multidevice rest
        ```
      </Accordion>

      <Accordion title="Update to latest version">
        ```bash
        # Pull latest image
        docker pull aldinokemal2104/go-whatsapp-web-multidevice:latest

        # Stop and remove old container
        docker stop whatsapp
        docker rm whatsapp

        # Start new container with same volume
        docker run --detach \
          --publish=3000:3000 \
          --name=whatsapp \
          --restart=always \
          --volume=whatsapp:/app/storages \
          aldinokemal2104/go-whatsapp-web-multidevice rest
        ```
      </Accordion>
    </AccordionGroup>

  <Tab title="Docker Compose">
    ### Docker Compose Installation

    Docker Compose is ideal for production deployments with better configuration management.

    <Steps>
      <Step title="Create docker-compose.yml">
        Create a `docker-compose.yml` file with your preferred configuration:

        <CodeGroup>
        ```yaml docker-compose.yml (Docker Hub)
        services:
          whatsapp:
            image: aldinokemal2104/go-whatsapp-web-multidevice
            container_name: whatsapp
            restart: always
            ports:
              - "3000:3000"
            volumes:
              - whatsapp:/app/storages
            command:
              - rest
              - --basic-auth=admin:admin
              - --port=3000
              - --debug=true
              - --os=Chrome
              - --account-validation=false

        volumes:
          whatsapp:
        ```

        ```yaml docker-compose.yml (GitHub Container Registry)
        services:
          whatsapp:
            image: ghcr.io/aldinokemal/go-whatsapp-web-multidevice
            container_name: whatsapp
            restart: always
            ports:
              - "3000:3000"
            volumes:
              - whatsapp:/app/storages
            command:
              - rest
              - --basic-auth=admin:admin
              - --port=3000
              - --debug=true
              - --os=Chrome
              - --account-validation=false

        volumes:
          whatsapp:
        ```

        ```yaml Environment Variables
        services:
          whatsapp:
            image: aldinokemal2104/go-whatsapp-web-multidevice
            container_name: whatsapp
            restart: always
            ports:
              - "3000:3000"
            volumes:
              - whatsapp:/app/storages
            environment:
              - APP_BASIC_AUTH=admin:admin
              - APP_PORT=3000
              - APP_DEBUG=true
              - APP_OS=Chrome
              - WHATSAPP_AUTO_REPLY=Thank you for your message
              - WHATSAPP_WEBHOOK=https://yourwebhook.com/handler
              - WHATSAPP_WEBHOOK_SECRET=super-secret-key
            command:
              - rest

        volumes:
          whatsapp:
        ```

        ```yaml Production (with .env)
        services:
          whatsapp:
            image: aldinokemal2104/go-whatsapp-web-multidevice
            container_name: whatsapp
            restart: always
            ports:
              - "3000:3000"
            volumes:
              - whatsapp:/app/storages
            env_file:
              - .env
            command:
              - rest

        volumes:
          whatsapp:
        ```
        </CodeGroup>
      </Step>

      <Step title="Create .env file (optional)">
        For environment variable configuration, create a `.env` file:

        ```bash .env
        # Application Settings
        APP_PORT=3000
        APP_HOST=0.0.0.0
        APP_DEBUG=false
        APP_OS=Chrome
        APP_BASIC_AUTH=admin:admin,user:pass123
        APP_BASE_PATH=
        APP_TRUSTED_PROXIES=0.0.0.0/0

        # WhatsApp Settings
        WHATSAPP_AUTO_REPLY=Thank you for your message. We'll get back to you soon.
        WHATSAPP_AUTO_MARK_READ=false
        WHATSAPP_AUTO_REJECT_CALL=false
        WHATSAPP_AUTO_DOWNLOAD_MEDIA=true
        WHATSAPP_WEBHOOK=https://yourwebhook.com/handler
        WHATSAPP_WEBHOOK_SECRET=super-secret-key
        WHATSAPP_WEBHOOK_EVENTS=message,message.ack,group.participants
        WHATSAPP_ACCOUNT_VALIDATION=true
        WHATSAPP_PRESENCE_ON_CONNECT=unavailable

        # Chatwoot Integration (optional)
        CHATWOOT_ENABLED=false
        CHATWOOT_URL=https://app.chatwoot.com
        CHATWOOT_API_TOKEN=your_token_here
        CHATWOOT_ACCOUNT_ID=12345
        CHATWOOT_INBOX_ID=67890
        CHATWOOT_DEVICE_ID=
        ```

        <Warning>
        Keep your `.env` file secure and never commit it to version control. Add it to `.gitignore`.
        </Warning>
      </Step>

      <Step title="Start services">
        ```bash
        docker-compose up -d
        ```

        View logs:
        ```bash
        docker-compose logs -f
        ```
      </Step>

      <Step title="Verify it's running">
        ```bash
        # Check status
        docker-compose ps

        # Test API
        curl http://localhost:3000/app/devices
        ```
      </Step>
    </Steps>

  <Tab title="Binary">
    ### Binary Installation

    Download pre-built binaries for your platform from GitHub releases.

    <Steps>
      <Step title="Install dependencies">
        Before running the binary, install required dependencies:

        <CodeGroup>
        ```bash Linux (Ubuntu/Debian)
        sudo apt update
        sudo apt install ffmpeg webp
        ```

        ```bash macOS
        brew install ffmpeg webp
        export CGO_CFLAGS_ALLOW="-Xpreprocessor"
        ```

        ```bash Windows (via WSL)
        # Use WSL2 with Ubuntu
        sudo apt update
        sudo apt install ffmpeg webp
        ```
        </CodeGroup>

        <Warning>
        Windows native is **not recommended**. Use WSL2 (Windows Subsystem for Linux) instead for best compatibility.
        </Warning>
      </Step>

      <Step title="Download the binary">
        Visit the [GitHub releases page](https://github.com/aldinokemal/go-whatsapp-web-multidevice/releases/latest) or download via command line:

        <CodeGroup>
        ```bash Linux AMD64
        # Replace v8.3.0 with the latest version
        wget https://github.com/aldinokemal/go-whatsapp-web-multidevice/releases/download/v8.3.0/whatsapp-linux-amd64
        chmod +x whatsapp-linux-amd64
        mv whatsapp-linux-amd64 whatsapp
        ```

        ```bash Linux ARM64
        wget https://github.com/aldinokemal/go-whatsapp-web-multidevice/releases/download/v8.3.0/whatsapp-linux-arm64
        chmod +x whatsapp-linux-arm64
        mv whatsapp-linux-arm64 whatsapp
        ```

        ```bash macOS Intel
        wget https://github.com/aldinokemal/go-whatsapp-web-multidevice/releases/download/v8.3.0/whatsapp-darwin-amd64
        chmod +x whatsapp-darwin-amd64
        mv whatsapp-darwin-amd64 whatsapp
        ```

        ```bash macOS Apple Silicon
        wget https://github.com/aldinokemal/go-whatsapp-web-multidevice/releases/download/v8.3.0/whatsapp-darwin-arm64
        chmod +x whatsapp-darwin-arm64
        mv whatsapp-darwin-arm64 whatsapp
        ```
        </CodeGroup>
      </Step>

      <Step title="Run the binary">
        <CodeGroup>
        ```bash Basic
        ./whatsapp rest
        ```

        ```bash With Options
        ./whatsapp rest \
          --port=3000 \
          --basic-auth=admin:admin \
          --debug=true \
          --os=Chrome
        ```

        ```bash With Webhook
        ./whatsapp rest \
          --webhook="https://yourwebhook.com/handler" \
          --webhook-secret="super-secret-key" \
          --autoreply="Thank you for your message"
        ```
        </CodeGroup>

        <Tip>
        Run `./whatsapp --help` to see all available options.
        </Tip>
      </Step>

      <Step title="Access the API">
        Open `http://localhost:3000` in your browser.
      </Step>
    </Steps>

    ### Running as a Service (Linux)

    For production deployments, run GOWA as a systemd service:

    <Accordion title="Create systemd service">
      1. Create service file:
      ```bash
      sudo nano /etc/systemd/system/whatsapp.service
      ```

      2. Add configuration:
      ```ini
      [Unit]
      Description=GOWA WhatsApp API
      After=network.target

      [Service]
      Type=simple
      User=www-data
      WorkingDirectory=/opt/whatsapp
      ExecStart=/opt/whatsapp/whatsapp rest --port=3000 --basic-auth=admin:admin
      Restart=always
      RestartSec=10

      [Install]
      WantedBy=multi-user.target
      ```

      3. Enable and start:
      ```bash
      sudo systemctl daemon-reload
      sudo systemctl enable whatsapp
      sudo systemctl start whatsapp
      ```

      4. Check status:
      ```bash
      sudo systemctl status whatsapp
      ```
    </Accordion>

  <Tab title="Build from Source">
    ### Build from Source

    Build GOWA from source for custom modifications or latest development version.

    <Steps>
      <Step title="Install prerequisites">
        <CodeGroup>
        ```bash Linux (Ubuntu/Debian)
        # Install Go 1.24.0 or higher
        wget https://go.dev/dl/go1.24.0.linux-amd64.tar.gz
        sudo rm -rf /usr/local/go
        sudo tar -C /usr/local -xzf go1.24.0.linux-amd64.tar.gz
        export PATH=$PATH:/usr/local/go/bin

        # Install dependencies
        sudo apt update
        sudo apt install ffmpeg webp git
        ```

        ```bash macOS
        # Install Go via Homebrew
        brew install go@1.24

        # Install dependencies
        brew install ffmpeg webp git
        export CGO_CFLAGS_ALLOW="-Xpreprocessor"
        ```
        </CodeGroup>

        Verify Go installation:
        ```bash
        go version  # Should show 1.24.0 or higher
        ```
      </Step>

      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/aldinokemal/go-whatsapp-web-multidevice.git
        cd go-whatsapp-web-multidevice/src
        ```
      </Step>

      <Step title="Build the binary">
        <CodeGroup>
        ```bash Linux & macOS
        go build -o whatsapp
        ```

        ```bash Windows (PowerShell)
        go build -o whatsapp.exe
        ```

        ```bash With optimizations
        go build -ldflags="-s -w" -o whatsapp
        ```
        </CodeGroup>

        <Tip>
        The `-ldflags="-s -w"` flag reduces binary size by stripping debug information.
        </Tip>
      </Step>

      <Step title="Run the binary">
        <CodeGroup>
        ```bash Linux & macOS
        ./whatsapp rest
        ```

        ```bash Windows
        .\whatsapp.exe rest
        ```
        </CodeGroup>
      </Step>

      <Step title="Access the API">
        Open `http://localhost:3000` in your browser.
      </Step>
    </Steps>

## Configuration Options

All installation methods support the same configuration options. See the [Configuration Guide](/guides/configuration) for detailed documentation.

### Command-Line Flags

<AccordionGroup>
  <Accordion title="Basic Options">
    ```bash
    --port=3000              # Server port (default: 3000)
    --host=0.0.0.0           # Host address (default: 0.0.0.0)
    --debug=true             # Enable debug logging
    --os="Chrome"            # Device name in WhatsApp
    --basic-auth=user:pass   # Basic authentication
    ```
  </Accordion>

  <Accordion title="WhatsApp Options">
    ```bash
    --autoreply="Message"            # Auto-reply message
    --auto-mark-read=true            # Mark incoming messages as read
    --auto-reject-call=true          # Reject incoming calls
    --auto-download-media=true       # Auto-download media files
    --presence-on-connect=unavailable # Presence: available|unavailable|none
    ```
  </Accordion>

  <Accordion title="Webhook Options">
    ```bash
    --webhook="https://webhook.com/handler"  # Webhook URL(s)
    --webhook-secret="secret"                # HMAC secret for webhooks
    --webhook-events="message,message.ack"   # Event whitelist
    --webhook-insecure-skip-verify=false     # Skip TLS verification
    ```
  </Accordion>

  <Accordion title="Advanced Options">
    ```bash
    --base-path="/gowa"              # Subpath deployment
    --account-validation=false       # Disable account validation
    --trusted-proxies="0.0.0.0/0"    # Trusted proxy IPs
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Docker: Permission denied">
    If you get permission errors with Docker volumes:

    ```bash
    # Fix volume permissions
    docker run --rm -v whatsapp:/data alpine chown -R 1000:1000 /data
    ```
  </Accordion>

  <Accordion title="macOS: CGO compilation error">
    On macOS, if you get CGO errors:

    ```bash
    export CGO_CFLAGS_ALLOW="-Xpreprocessor"
    ```

    Add this to your `~/.zshrc` or `~/.bash_profile` to make it permanent.
  </Accordion>

  <Accordion title="Binary: FFmpeg not found">
    Ensure FFmpeg is installed and in your PATH:

    ```bash
    # Check FFmpeg
    ffmpeg -version

    # Linux: Install FFmpeg
    sudo apt install ffmpeg

    # macOS: Install FFmpeg
    brew install ffmpeg
    ```
  </Accordion>

  <Accordion title="Port already in use">
    Change the port using `--port` flag or `APP_PORT` environment variable:

    ```bash
    ./whatsapp rest --port=8080
    ```
  </Accordion>

  <Accordion title="Cannot connect to WhatsApp servers">
    - Check your internet connection
    - Verify firewall settings allow outbound HTTPS (port 443)
    - Try reconnecting: `curl http://localhost:3000/app/reconnect`
    - Check logs for errors
  </Accordion>
</AccordionGroup>
